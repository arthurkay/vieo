1. Overview

vieo is a self-hosted video streaming platform that takes your media sources (files, URLs, live streams) and converts them into web-friendly HLS format for playback in any modern browser.

What can vieo do?

  • Transcode video files — Convert MP4, MKV, and other formats to adaptive HLS streaming
  • Stream live sources — Ingest RTMP, RTSP, and HLS streams and make them playable in the browser
  • Live preview — Watch streams in real-time as they are being transcoded
  • Organize with channels — Group related streams into channels for easy management
  • Monitor everything — Real-time progress tracking, disk usage monitoring, and job management

2. Getting Started

1 Open vieo in your browser
Navigate to http://localhost:8080 (or the address your administrator configured).
2 Create your first channel
Go to Channels in the sidebar and click New Channel. Give it a name (e.g., "My Streams") and an optional description.
3 Add a source
Open your channel and click Add Source. Select the source type and enter the URL or file path.
4 Start transcoding
Click Start Transcoding on your source. The job will appear on the Jobs page with a real-time progress bar.
5 Watch your stream
Once transcoding starts, click the Play button on the job to open the player. You can watch while transcoding is still in progress.
Tip: The player works while transcoding is running — you don't need to wait for it to finish. Early segments become available within seconds.

3. Managing Channels

Channels are organizational containers for your media sources. Think of them as folders or categories.

Creating a channel

  1. Click Channels in the sidebar
  2. Click New Channel
  3. Enter a Name (required) — this is what appears in the UI
  4. Enter a Slug (required) — a URL-friendly identifier (e.g., my-streams)
  5. Enter a Description (optional)
  6. Click Create

Editing a channel

  1. Click Channels in the sidebar
  2. Click the Edit (pencil) icon on the channel card
  3. Modify the name, slug, or description
  4. Click Update

Deleting a channel

  1. Click Channels in the sidebar
  2. Click the Delete (trash) icon on the channel card
Warning: Deleting a channel also deletes all sources, outputs, and jobs associated with it. This action cannot be undone.

4. Adding Sources

A source represents a media input — a file, URL, or live stream that vieo will transcode.

Adding a source to a channel

  1. Open a channel by clicking View on a channel card
  2. Click Add Source
  3. Select the Type:
    • File / URL — For local files or HTTP URLs
    • HLS Stream — For HLS master playlists (.m3u8)
    • RTMP — For RTMP live streams
    • RTSP — For RTSP camera streams
    • Device — For local capture devices
  4. Enter the URL or file path
  5. Click Add
Note: The stream type (audio+video, video-only, audio-only) is automatically detected when you start transcoding. You don't need to specify it manually.

5. Transcoding Streams

Transcoding converts your source media into HLS format — a sequence of small .ts video segments and an .m3u8 playlist file that browsers can stream.

Starting a transcode

  1. Open a channel and find your source
  2. Click Start Transcoding
  3. vieo will:
    1. Create an output directory for the HLS files
    2. Probe the source to determine codec, duration, and stream type
    3. Begin transcoding with ffmpeg
  4. The job appears on the Jobs page with a progress bar

What happens during transcoding?

  • Probing — vieo analyzes the source to detect video/audio streams, resolution, and duration
  • Segment creation — ffmpeg splits the output into 4-second .ts segments
  • Playlist updates — The .m3u8 playlist is updated as each segment completes
  • Progress tracking — Real-time progress is shown based on the output timestamp vs. total duration

Output format

ContainerMPEG-TS (.ts segments)
Video codecH.264 (libx264, fast preset, CRF 23)

6. Watching Streams

vieo includes a built-in web player powered by hls.js that works in all modern browsers.

Opening the player

  • From the Jobs page: Click the Play icon on any running or completed job
  • From the Dashboard: Click on any stream thumbnail in the "Live Streams" section
  • From Channel Detail: Click View Stream on a source with an active output

Live preview (during transcoding)

You can watch a stream while it's still being transcoded. The player automatically switches to live mode when viewing a running job:

  • The player buffers a few segments ahead and plays from the most recent content
  • If a segment isn't written yet, the player retries automatically
  • Once transcoding completes, the player switches to on-demand mode for full seeking
Tip: For the best live preview experience, wait at least 8-12 seconds after starting a transcode before opening the player. This allows a few segments to be written.

Player controls

ControlDescription

Note: The player page is independent from the rest of the UI — navigating away from it (e.g., to check job progress) won't interrupt playback if you use the browser's back button.

7. Managing Jobs

A job represents a single transcoding task — converting one source into HLS output.

Job statuses

StatusDescriptionActions available
PendingWaiting for a transcoding slotStop, Delete
RunningCurrently transcodingPause, Stop, Play, Delete
PausedTemporarily suspended (disk low or manual)Stop, Retry, Delete
CompletedTranscoding finished successfullyPlay, Retry, Delete
FailedTranscoding error occurredRetry, Delete
StoppedManually stopped by userRetry, Delete

Job actions

  • Pause — Temporarily suspends transcoding. The job can be resumed later from where it left off.
  • Stop — Stops the job permanently. The output is kept but marked as stopped.
  • Retry — Restarts the job from the beginning. Existing output files are cleaned before restarting.
  • Delete — Removes the job record from the database.

Filtering jobs

Use the status dropdown at the top of the Jobs page to filter by status (e.g., show only "Running" jobs).

Progress tracking

Running and paused jobs display a progress bar showing the current transcoding progress as a percentage. Progress is updated in real-time via WebSocket.

9. Troubleshooting

Stream won't play

  • Check the job status — If it shows "Failed", the error message will explain why
  • Wait for segments — If transcoding just started, wait 10-15 seconds for the first segments to be written
  • Check the URL — Make sure the source URL is accessible from the server
  • Browser compatibility — The player works in Chrome, Firefox, Safari, and Edge

Transcoding is slow

  • CPU-bound — Video transcoding is CPU-intensive. Faster CPUs produce faster results
  • Source format — Some formats (e.g., ProRes, RAW) require more processing than others
  • Resolution — Higher resolution sources take longer to transcode

Disk usage warnings

  • Warning (90%) — Running jobs are paused automatically to prevent disk filling up
  • Critical (95%) — All jobs are force-stopped
  • Recovery — Jobs resume automatically when disk usage drops below the warning threshold
  • Manual cleanup — Delete old jobs and their output to free space

Job stuck in "Pending"

  • Max jobs reached — vieo limits concurrent transcoding jobs (default: 3). Wait for another job to finish
  • Resume on restart — Pending/running/paused jobs automatically resume when the server restarts

Source probe fails

  • Network issues — The server must be able to reach the source URL
  • Timeout — Remote HLS streams may take longer to probe (up to 60 seconds)
  • Format support — Ensure ffmpeg supports the input format
